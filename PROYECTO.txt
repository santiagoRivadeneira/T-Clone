  TWITTER CLONE — "The Flock" Challenge
  Resumen del proyecto
═══════════════════════════════════════════════════════════════

STACK
─────
Frontend : React 18 + Vite + TanStack Query + React Router v6
           Tailwind CSS + shadcn/ui + Lucide Icons
Backend  : Fastify + TypeScript + Prisma ORM + PostgreSQL
           JWT (RS256) + Zod (validación) + Vitest + Supertest

═══════════════════════════════════════════════════════════════
  FUNCIONALIDADES IMPLEMENTADAS
═══════════════════════════════════════════════════════════════

AUTENTICACIÓN
  ✓ Registro con username, email, password (hash bcrypt)
  ✓ Login con JWT de 7 días
  ✓ Perfil propio (GET /auth/me, PATCH /auth/me)
  ✓ Actualización de displayName, bio, avatarUrl

TWEETS
  ✓ Crear tweet (hasta 280 caracteres, sin espacios vacíos)
  ✓ Eliminar tweet propio (403 si es ajeno)
  ✓ Responder a un tweet (replyToId)
  ✓ Vista detallada con lista de replies
  ✓ Paginación por cursor en todas las listas

INTERACCIONES (todas idempotentes — nunca retornan 409)
  ✓ Like / Unlike     POST/DELETE /tweets/:id/like
  ✓ Retweet / Undo    POST/DELETE /tweets/:id/retweet
  ✓ Bookmark / Undo   POST/DELETE /tweets/:id/bookmark
  ✓ Lista de bookmarks paginada (GET /tweets/bookmarks/list)

USUARIOS
  ✓ Perfil público con contador de seguidores/siguiendo/tweets
  ✓ Búsqueda por username o displayName (con límite de 100 chars)
  ✓ Follow / Unfollow (idempotente, guard contra auto-follow)
  ✓ Lista de seguidores y seguidos
  ✓ Flag isFollowing en perfil y búsqueda (cuando autenticado)

TIMELINE
  ✓ Timeline de tweets de usuarios seguidos (cursor pagination)
  ✓ Solo tweets top-level (no replies)

NOTIFICACIONES
  ✓ Creación automática al: dar like, retweetear, responder, seguir
  ✓ Guard: no se notifica al propio autor de la acción
  ✓ Lista paginada con actor + tweet
  ✓ Contador de no leídas (GET /notifications/unread-count)
  ✓ Marcar todas como leídas (PATCH /notifications/read)
  ✓ Badge de no leídas en sidebar (se actualiza en tiempo real)

TIEMPO REAL (Server-Sent Events)
  ✓ Una sola conexión SSE por sesión (SSEContext compartido)
  ✓ new-tweet: aparece banner en timeline para mostrar nuevos tweets
  ✓ delete-tweet: se elimina del timeline sin recargar
  ✓ new-notification: incrementa badge de notificaciones en sidebar
  ✓ Keep-alive ping cada 25 s para evitar timeout de proxies
  ✓ Reconexión automática del cliente (retry: 3000)

UI / UX
  ✓ Diseño dark mode al estilo Twitter/X
  ✓ Sidebar de navegación (desktop) + bottom nav (mobile)
  ✓ Infinite scroll estable con IntersectionObserver
  ✓ Optimistic updates con rollback en likes, retweets, bookmarks
  ✓ Avatar con fallback a inicial del displayName
  ✓ Paginación por cursor en timeline, perfil, bookmarks, notifs

  COSAS CONOCIDAS / NO IMPLEMENTADAS (fuera de alcance)

═══════════════════════════════════════════════════════════════
  COMANDOS ÚTILES
═══════════════════════════════════════════════════════════════

  # Backend
  cd backend
  npm run dev          ← inicia servidor en :3000
  npm run db:migrate   ← aplica migraciones Prisma
  npm run db:generate  ← regenera Prisma Client
  npm run db:seed      ← datos de prueba
  npm test             ← corre todos los tests

  # Frontend
  cd .. (raíz)
  npm run dev          ← inicia Vite en :5173
